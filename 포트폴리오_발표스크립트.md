# 포트폴리오 3분 발표 스크립트

안녕하세요. 이 포트폴리오는 알라딘 업무자동화 포지션에 맞춰 만든 `B2B 주문/정산 업무 자동화 프로토타입`입니다.

먼저 문제 정의부터 말씀드리겠습니다. 운영 업무에서는 CSV나 엑셀로 받은 주문 데이터를 확인하고, 재고나 배송지 문제를 찾고, 정산 금액을 계산한 뒤 담당자에게 공유하는 반복 업무가 자주 발생한다고 가정했습니다. 이런 업무는 사람이 매번 처리하면 시간이 오래 걸리고 누락이나 계산 실수가 생길 수 있습니다.

그래서 저는 이 업무를 `주문 확인`, `상태 분류`, `정산 계산`, `예외 건 공유` 네 단계로 나눴습니다. 이 중 규칙이 명확한 부분은 자동화하고, 사람이 판단해야 하는 부분은 확인 필요 상태로 분리했습니다.

프로토타입에서는 샘플 주문 데이터를 기준으로 재고가 충분하면 출고 가능, 재고가 부족하거나 배송지 확인이 필요하면 확인 필요, 재고가 없으면 보류로 자동 분류합니다. 그리고 총 주문금액, 출고 가능 건수, 확인 필요 건수를 계산한 뒤 현업 담당자에게 공유할 메시지까지 자동으로 생성합니다.

이 포트폴리오의 핵심은 완성된 대형 시스템을 만든 것이 아니라, 현업과 빠르게 대화하고 검증할 수 있는 작은 자동화 결과물을 만들었다는 점입니다. 실제 업무에서는 먼저 CSV 기반으로 검증하고, 반복 사용 가치가 확인되면 내부 운영툴이나 API 연동으로 확장할 수 있습니다.

제가 이 포지션에서 강조하고 싶은 강점은 세 가지입니다. 첫째, 업무를 바로 코드로 만들기보다 먼저 흐름과 예외를 구조화하겠습니다. 둘째, 엑셀, CSV, Python 또는 JavaScript, AI 도구를 활용해 빠르게 프로토타입을 만들겠습니다. 셋째, 자동화 가능한 업무와 프로덕션 개발이 필요한 업무를 구분해서 현실적인 개선안을 제안하겠습니다.

입사 후에는 현업 담당자들의 반복 업무를 먼저 듣고, 빈도와 오류 위험이 높은 업무부터 작은 자동화로 개선해보고 싶습니다.
