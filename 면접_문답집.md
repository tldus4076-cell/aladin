# 알라딘 업무자동화 포지션 면접 문답집

## 1. 자기소개를 해주세요.

답변 방향:
저는 반복 업무를 구조화하고 자동화 가능한 단위로 쪼개는 데 관심이 있습니다. 현업 담당자가 실제로 겪는 불편을 먼저 듣고, 엑셀/CSV/간단한 스크립트/AI 도구를 활용해 빠르게 검증 가능한 프로토타입을 만드는 방식으로 기여하고 싶습니다.

## 2. 이 포지션에 지원한 이유는 무엇인가요?

답변 방향:
채용공고에서 개발보다 먼저 현업 프로세스를 이해하고 자동화 가능 영역을 찾는 역할이 강조되어 있어서 적합하다고 느꼈습니다. 단순히 코드를 만드는 것이 아니라 운영 업무의 시간을 줄이고 오류를 줄이는 결과를 만드는 점이 매력적입니다.

## 3. 현업 업무를 자동화할 때 어떤 순서로 접근하나요?

답변 방향:
먼저 업무 흐름을 듣고 입력 데이터, 처리 규칙, 예외 상황, 결과물을 정리합니다. 그 다음 반복 빈도와 오류 위험이 높은 업무를 우선순위로 잡습니다. 이후 CSV나 엑셀 기반으로 작은 프로토타입을 만들고, 현업 피드백을 받아 API 연동이나 운영툴 반영이 필요한지 판단합니다.

## 4. 자동화하면 안 되는 업무는 어떻게 구분하나요?

답변 방향:
규칙이 자주 바뀌거나 사람의 판단이 핵심인 업무는 완전 자동화보다 체크리스트나 보조 도구가 적합하다고 봅니다. 반대로 입력과 판단 기준이 명확하고 반복 빈도가 높은 업무는 자동화 우선순위가 높습니다.

## 5. AI 도구는 실무에서 어떻게 활용할 건가요?

답변 방향:
요구사항 정리, 코드 초안 작성, 테스트 케이스 생성, 예외 상황 점검에 활용하겠습니다. 다만 고객 정보, 거래처 정보, 내부 운영 데이터는 그대로 입력하지 않고 비식별화하거나 샘플 데이터로 바꿔 사용하겠습니다.

## 6. Python과 JavaScript 중 무엇을 쓸 건가요?

답변 방향:
CSV/엑셀 데이터 처리와 대량 집계는 Python pandas가 적합하고, 브라우저에서 바로 보여주는 간단한 프로토타입은 JavaScript가 적합합니다. 업무 성격에 따라 도구를 고르겠습니다.

## 7. API 연동 경험이 부족하면 어떻게 대응할 건가요?

답변 방향:
API는 요청, 응답, 인증, 에러 처리 구조를 먼저 이해하겠습니다. 초기에는 Postman이나 간단한 스크립트로 응답을 확인하고, 운영에 반영하기 전에는 실패 케이스와 재시도 기준을 정리하겠습니다.

## 8. 이 포트폴리오의 핵심은 무엇인가요?

답변 방향:
B2B 주문/정산 업무에서 반복되는 CSV 확인, 상태 분류, 정산 합계, 예외 메시지 작성을 자동화 흐름으로 묶은 것입니다. 완성된 대형 시스템이 아니라 현업과 빠르게 검증하기 위한 프로토타입이라는 점이 핵심입니다.

## 9. 현업 담당자와 의견이 다르면 어떻게 하나요?

답변 방향:
먼저 담당자가 왜 그렇게 판단하는지 업무 맥락을 확인하겠습니다. 자동화는 현업의 일을 대체하는 것이 아니라 반복 부담을 줄이는 것이므로, 실제 사용자가 납득할 수 있는 기준을 함께 정리하겠습니다.

## 10. 자동화 우선순위는 어떻게 정하나요?

답변 방향:
반복 빈도, 소요 시간, 오류 발생 가능성, 고객/정산 영향도, 규칙의 명확성, 개발 난이도를 기준으로 정하겠습니다. 효과는 큰데 구현이 쉬운 업무부터 빠르게 처리하는 것이 좋다고 생각합니다.

## 11. 프로토타입과 프로덕션 개발의 차이는 무엇인가요?

답변 방향:
프로토타입은 문제 해결 가능성과 업무 흐름을 빠르게 검증하는 목적입니다. 프로덕션 개발은 안정성, 보안, 권한, 로그, 장애 대응, 유지보수까지 고려해야 합니다. 저는 먼저 프로토타입으로 가치를 확인한 뒤 프로덕션 개발이 필요한 범위를 분리하겠습니다.

## 12. 자동화 도입 후 성과는 어떻게 측정하나요?

답변 방향:
업무 처리 시간 감소, 수작업 입력 횟수 감소, 오류 건수 감소, 예외 대응 시간 단축, 담당자 만족도 같은 지표로 측정하겠습니다. 자동화 전후 시간을 비교하면 효과를 명확히 볼 수 있습니다.

## 13. 개인정보나 내부 데이터를 다룰 때 주의할 점은 무엇인가요?

답변 방향:
필요한 데이터만 사용하고 접근 권한을 제한해야 합니다. AI 도구에는 실제 개인정보나 내부 민감 데이터를 넣지 않으며, 로그와 파일 저장 위치도 관리해야 합니다.

## 14. 입사 후 처음 한 달 동안 어떻게 일하고 싶나요?

답변 방향:
먼저 운영/백오피스/B2B 담당자들의 반복 업무를 듣고 업무 흐름을 문서화하겠습니다. 이후 자동화 후보를 우선순위화하고, 빠르게 만들 수 있는 작은 프로토타입부터 검증하겠습니다.

## 15. 마지막으로 하고 싶은 말이 있나요?

답변 방향:
저는 거창한 시스템보다 실제 담당자가 매일 쓰는 작은 자동화가 업무 효율을 크게 바꿀 수 있다고 생각합니다. 알라딘의 운영 업무를 이해하고 빠르게 실험하며 개선하는 사람으로 기여하고 싶습니다.
